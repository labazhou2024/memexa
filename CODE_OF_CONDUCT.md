# Community guidelines

This project adopts the **Contributor Covenant, version 2.1**, by
reference. The canonical text is published by its authors at
<https://www.contributor-covenant.org/version/2/1/code_of_conduct/>.
The text below summarises what that means in practice for `memexa`; the
upstream document controls if anything here is unclear.

## In short

Treat everyone who shows up in this repo, in PRs, in Discussions, or in
linked chat rooms as a colleague. Disagree with ideas, not with people.
Assume good faith. When the maintainer asks you to stop a pattern of
behavior, stop it.

## Scope

These guidelines apply to every space owned by the project: the GitHub
repository, the issue tracker, pull requests, Discussions, and any
official chat rooms or events the maintainer sets up. They apply to
both contributors and the maintainer.

## Reporting

If someone's behavior is making it hard for you to participate, open a
private report through GitHub's Security Advisory channel for this
repo. The maintainer will read it and respond within seven days. You
will be told what action was taken; people who report concerns will not
be identified to the subject of the report without their consent.

If the issue is with the maintainer themselves, escalate to GitHub
Trust & Safety: <https://docs.github.com/en/communities/maintaining-your-safety-on-github>.

## Consequences

Possible responses, depending on severity and pattern:

1. Private clarification: the maintainer explains what was off and asks
   for a change going forward.
2. Public clarification: the maintainer publicly states the behavior
   was not acceptable, with no further sanction.
3. Temporary cool-off: blocked from the repo for a defined period.
4. Permanent ban: blocked from the repo indefinitely.

The maintainer alone decides which response fits. The decision can be
appealed once, in writing, and the appeal goes to the same channel.
