# Case Study 02 · 5 分钟见面简报模板

[English](02_meeting_brief_pattern.md) · **中文**

> **问题一句话**: 你 30 分钟后要见 X。基线是什么? 上次说啥? 开放话题
> 在哪? 雷区在哪? 大多数人靠翻聊天记录 — 慢且容易错。memex 把它做成
> 5 分钟产出 4 段简报。

## 受众

- **办事人 (运维风)** — 会前 / 1-on-1 前简报
- **科研/学生** — 见导师 / 合作者 / 同门前
- **PM / sales** — 任何客户 call 前

同一个模式, 不同词汇。

## 4 段 brief 模板

```
   ┌────────────────────────────────────────────────────────┐
   │ §1  BASELINE 基线                                       │
   │     对方是谁, 你怎么认识, 关键时间锚                    │
   │     "一句话画像"                                         │
   ├────────────────────────────────────────────────────────┤
   │ §2  LAST CONTACT 上次联系                                │
   │     什么时候 / 说了啥 / 什么调性                         │
   │     "对方记得的你"                                       │
   ├────────────────────────────────────────────────────────┤
   │ §3  OPEN THREADS 开放话题                                │
   │     未办的承诺 / 还在等的回答                            │
   │     "对方期待你主动提的事"                                │
   ├────────────────────────────────────────────────────────┤
   │ §4  LANDMINES 雷区                                       │
   │     敏感话题 / 不要碰的事                                │
   │     "什么不要说"                                          │
   └────────────────────────────────────────────────────────┘
```

模板**有指定结构**。别加段。别删段。重点是你 5 分钟能填完, 冷用也行。

## 3 命令抽取流水线

```
                      ┌────────────────────────────┐
                      │ 输入: 人名 / 邮箱           │
                      └─────────────┬──────────────┘
                                    │
              ┌─────────────────────┼─────────────────────┐
              ▼                     ▼                     ▼
       ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
       │  memex arc   │     │ memex quick  │     │ memex quick  │
       │  (60 卡,     │     │ "<名> 这周"  │     │ "<名> 问"    │
       │  关系全史)   │     │ (近因)       │     │ (开放诉求)   │
       └──────┬───────┘     └──────┬───────┘     └──────┬───────┘
              │                    │                    │
              └────────────────────┼────────────────────┘
                                   │
                                   ▼
                      ┌────────────────────────────┐
                      │ 人工合成 (4 min, 填模板)    │
                      └────────────────────────────┘
```

## Demo: 用 `demo_dataset` 给 Alice 写 brief

你明天早上见 Alice。3 个查询:

### 查询 1 — `arc` 拿关系广度

```bash
memex arc "Alice" --max-cards 60
```

你从上到下读, 抽出:

- Alice 是组会牵头人 (反复出现的 "组会改到..." 消息)
- DDIA 读书搭子 (推荐 Ch.5 replication, 2024-01-05)
- 她做 ppt; 你做第 3 部分; Bob / Carol 做数据
- 沟通主要 QQ 1-on-1 + 微信群

→ §1 BASELINE 填完。

### 查询 2 — `quick` 拿最近 7 天

```bash
memex quick "Alice 1月" --max-k 20
```

- 2024-01-22: 她把组会推迟到 2024-01-30 (节假日)
- 2024-01-15: 她准点 14:00 开了组会
- 2024-01-12: 她说 "ppt 做好, 等 Bob 数据合稿"

→ §2 LAST CONTACT 就是 2024-01-22 的延期通知。

### 查询 3 — `quick` 拿开放诉求

```bash
memex quick "Alice 等 待 问" --max-k 15
```

这个语义会抓住 "等" / "需要" / "能不能" 这些询问/期待语义。

- 2024-01-12 qq: Alice 说 "等 Bob 数据合稿" — 但 Bob 数据 2024-01-09
  18:03 就交了。**开放话题: 确认她有没有拿到 Bob 数据。**

→ §3 OPEN THREADS = "确认 Alice 有没有 Bob 的最终数据; 否则中期报告
   合稿就卡住"。

§4 LANDMINES — 关系里有明显紧张吗? 扫 `arc` 输出找负面语气标记。本 demo
数据里没大摩擦。(真实数据里要找: 之前取消的安排, 批评话, 钱的话题。)

## 填好的 brief

```markdown
# 见面 brief: Alice — 2024-01-23 早

## §1 基线 (她是谁)
- 组会牵头人; DDIA 读书搭子
- 负责: 排期 + 最终 ppt 合稿
- 分工: Alice=ppt, 我=section 3, Bob=ppt 第 5 页, Carol=实验数据
- 沟通渠道: QQ 1-on-1 + 微信群

## §2 上次联系
- 2024-01-22 — 她把组会推到 2024-01-30 (节假日)
- 之前: 2024-01-15 当面开会, 正常节奏

## §3 开放话题
- 她到底拿到 Bob 的最终数据了吗? 她 1-12 说 "等 Bob 数据合稿",
  但 Bob 是 1-09 交的。可能 sync 出问题了。
- 中期报告合稿状态

## §4 雷区
- 聊天里没看到敏感点
- Note: 不要主动提节假日安排 (是她做的延期决定)
```

5 分钟从 "memex arc Alice" 到填好 brief。你带着已经扫过整个关系的状态
进会议 — 否则要花 30 分钟翻聊天记录。

## 为啥就这个模板?

每段对应对方默默问的一个问题:

| Section | 对方暗中问的 | 跳过会怎样 |
|---|---|---|
| §1 基线 | "你真的认识我吗?" | 你犯新手错 (忘了她角色) |
| §2 上次联系 | "你记得我们说过啥吗?" | 你重新讨论已经达成的事 |
| §3 开放话题 | "你打算回应我问过的事吗?" | 对方觉得被忽视 |
| §4 雷区 | "你会不会读空气?" | 你踩到痛处 |

任何一段跳过, 都是半盲进会议。

## 何时跳步 (vs 完整 3 步)

| 场景 | 要 3 步全跑? |
|---|---|
| 每天见的同事, 昨天刚见过 | 不要。`quick "<名> 今天"` 就够 |
| 多月固定 1-on-1 | 跳过 §1。`arc` 跑一次存基线; 之后只刷 §2-§4 |
| 几乎不熟的人 | 要。完整跑 + §1 多花时间 |
| 3+ 月不联系后再聚 | 要, **且**重点查 §4 — 关系会漂移 |

## 包装成函数

模式内化后, 包成 shell 函数:

```bash
# ~/.bashrc 或 PowerShell profile
function brief {
  echo "=== §1 基线 ==="
  memex arc "$1" --max-cards 60
  echo ""
  echo "=== §2 最近 7 天 ==="
  memex quick "$1 最近" --max-k 15
  echo ""
  echo "=== §3 开放话题 ==="
  memex quick "$1 等 待 问 还没" --max-k 15
}

# 用法
brief "Alice" > today_alice.md
```

会议时 today_alice.md 开在侧边窗。

## 用到你自己的数据

模板通用, 查询换一个词:

```bash
brief "<你的对手方>"
```

有人把这些 brief 用 git 管 (私有仓), 之后能回看 "我当初对 X 的判断是啥?"。

## 何时这个不适用

- **从未联系过对方** — memex 没数据。用 LinkedIn / 简历代替。
- **纯信息会议 (听课/讲座)** — 不需关系层。跳 §3-§4, 用
  [03_project_status_check.zh.md](../../examples/demo_dataset/walkthroughs/03_project_status_check.zh.md)
  做主题准备。
- **对抗 / 谈判** — 4 段模板太友好。加 §5 "对方利益" + §6 "BATNA"
  (本文不涵盖)。
